# my-coach-app 要件定義

## アプリ概要

中学生向けの学習支援チャットアプリ。
問題の答えをそのまま教えるのではなく、考え方・解き方をAIがコーチング形式で教える。

---

## ユーザー

- 主な利用者：中学生（2名限定）
- 利用シーン：宿題や試験勉強で問題が解けないとき
- 公開範囲：特定のユーザーのみ（不特定多数への公開なし）

---

## 技術スタック

| 項目 | 採用技術 |
|---|---|
| フレームワーク | Next.js (App Router) |
| AIプロバイダー | Google Gemini 2.0 Flash（無料枠） |
| ホスティング | Vercel（Hobby プラン・無料） |
| 認証 | NextAuth.js v5 + Google OAuth |
| スタイリング | Tailwind CSS + shadcn/ui |
| AI呼び出し | Route Handler (`/api/chat`) + ReadableStream によるストリーミング |
| システムプロンプト | `lib/system-prompt.ts` に定数として管理 |
| Markdownレンダリング | `react-markdown` |

---

## 機能要件

### 認証
- Googleアカウントでログイン・ログアウトできる
- 未ログイン状態ではチャットを利用できない
- `ALLOWED_EMAILS` 環境変数に許可するGmailアドレスを列挙し、NextAuthの `signIn` コールバックで弾く
- 許可されていないアカウントでログインした場合はNextAuthのデフォルトエラーページを表示する

### チャット
- テキストで問題や質問を入力できる
- Enterキーで送信、Shift+Enterで改行
- AIが返答する（答えを教えるのではなく、ヒント・考え方を段階的に提示）
- 繰り返し対話できる（マルチターン）
- 対応教科は全教科（数学・理科・英語・国語・社会など）
- AIの返答はストリーミングでリアルタイム表示する
- 送信後、空のAIメッセージバブルを表示してからテキストが流れ込む形式
- AIの返答はMarkdownレンダリングする（太字・箇条書き・コードブロックなど）
- Geminiの無料枠上限に達した場合は「しばらく時間をおいてから試してね」などの親切なメッセージを表示する

### 会話履歴・セッション管理
- 直近3セッション分をlocalStorageに保存する
- ログイン後は空の新しいセッションが自動的に始まる（すぐ入力できる）
- メッセージが0件のセッションはlocalStorageに保存しない
- 「新しい会話」ボタンで明示的に新セッションを開始する
- 過去セッションはドロワー（引き出し型メニュー）から参照できる
- ドロワーの各セッションには最初のユーザーメッセージの冒頭（30文字程度）を表示する
- 4セッション目を開始すると最古のセッションが削除される
- AIには現在のセッションの全履歴のみ送信する（セッションをまたいだ文脈は引き継がない）

## 非機能要件

- 無料または低コストで運用できること（Gemini 無料枠 + Vercel 無料枠）
- 日本語でのやりとりに対応していること
- 中学生が使いやすいシンプルなUI
- スマートフォン対応（レスポンシブデザイン）

---

## 画面構成

### トップページ
- アプリの説明と「Googleでログイン」ボタンのみ表示

### チャット画面
- ログイン後に表示
- メッセージの入力欄と送信ボタン（Enter送信 / Shift+Enter改行）
- 自分の発言とAIの返答を交互に表示（ストリーミング）
- ドロワーメニュー：「新しい会話」ボタン + 過去3セッション一覧（各セッションは最初のメッセージ冒頭で識別）
- ログアウトボタン

---

## AIの振る舞い方針

- 答えをすぐに教えない
- 問題の種類を把握し、必要な知識を確認する
- ヒントを1つずつ出して、生徒が自分で考えられるよう誘導する
- 生徒が答えを求めてきても、最後の一手は生徒自身に言わせる形で誘導する（答えの直前まで導く）
- 間違えたら優しく訂正し、正しい考え方を示す
- フレンドリーで親しみやすい口調で話す（「〜だね」「〜してみよう！」）
- 励ましの言葉を添えて学習意欲を高める
